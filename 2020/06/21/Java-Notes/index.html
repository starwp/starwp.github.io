The first snippet is about the type of math operation.

long a = 100000000L;
short b = 10;
byte c = 8;
System.out.print(a + b);// should be a long type
System.out.print(b + c);// should be an int type

Then, we take a look at the operation i++ and ++i.

int a = 1;
int b = a++;
int c = ++a;
System.out.print(b);// should output 1  
System.out.print(c);// should output 3

For the string, it has a difference depends where we put it.

int a = 10;
int b = 20;
System.out.print('' + a + b);// should output string 1020 
System.out.print(a + b + '');// should output string 30

抽象类中不一定有抽象方法。但是抽象方法必须出现在抽象类。

interface: 是类的蓝图 (契约)。多个interface, interface实现方法，

接口支持多继承，接口只包括两部分内容，一部分：常量。一部分：抽象方法 (abstract method， 因此可以省略public abstract关键字)。接口中的方法，不能带有方法体。

接口是完全抽象的。

java 向上转型，向下转型 java upcasting， java down casting

编辑阶段：base class type 静态绑定method， static binding

运行阶段：derived class type动态绑定，dynamic binding method, 所以引入overriding.

  • At compile time, usually the base class type is used to reference an object,
  • though at run time this object could be of a derived type, so when an overridden method is called, the implementation that is called is dependent on what kind of object is doing the calling (base vs. a derived type) which is unknown at compile time.

Overridden functions are functions that have the same signature, but are implemented in different derived classes

Overloading (not really polymorphism) is simply multiple functions which have the same name but different signatures (think multiple constructors for an object taking different numbers of arguments). Which method is called is known at compile time, because the arguments are specified at this time.

私有方法无法override, 覆盖

虽然有接口，但是抽象类的作用是其中可以写非抽象方法，以达到代码复用的目的，所以抽象类不可或缺。

把一组数据结构和处理他们的方法组成对象（object），把相同行为的对象归纳为类（class），通过封装（encapsulation）隐藏细节，通过继承（inheritance）实现类的generalization, 通过多态（polymorphism）实现动态分派dynamic dispatch

2 Memory Management

The local primitive data type including int, float, char and others (8 in total) are stored in the stack memory. For example,

int a = 10; // the type, name, value of the variable is stored in the stack memory.

In addition, the local variables which are all used to store the references of the objects. These variables are also stored in the stack memory. However, the objects are stored in the heap memory. For instance,

Integer a = new Integer();// the type, name, represented reference of the variable a is stored in the stack memory, but key word new makes the oject Integer() in the heap memory.

In addition, for the heap memory, there exists

  1. Young Generation
  2. Old Generation
  3. Permanent Generation

Notice that the string objects (if not declared by new) should be stored in the string pool in the permanent memory in the heap, which causes the following:

String a = 'abc';
String b = 'abc';
System.out.println(a==b);//should return True

String c = new String('abc');
String d = new String('abc');
System.out.println(a==b);//should return False

Please be careful that ‘’==’’ is a reference matching operator if the variables are not primitive data types.
